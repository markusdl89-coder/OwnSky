### 🗃️ ПАСПОРТ МОДУЛЯ: database.py (Слой данных)

* **Роль в игре:** Переводчик между ботом (на Render) и базой данных (на Neon). Сам код крутится на хостинге, но все запросы шлет в облако.
* **Связь с базой:** Библиотека `pg8000`, использует переменную среды `DATABASE_URL`.
* **Функции для подключения из других слоев (API модуля):**
  * `get_player_data(user_id)` -> Возвращает словарь с текущими параметрами капитана.
  * `update_inventory(owner_type, owner_id, ...)` -> Изменяет количество/цену товаров в трюме корабля или на складе города.
  * `register_new_player(user_id, username)` -> Создает новую запись игрока в Neon при первой регистрации.
### 📊 СХЕМА СТРУКТУРЫ ТАБЛИЦ В NEON DB

1. **Таблица `players` (Капитаны)**
   * `user_id` (BIGINT, PRIMARY KEY) — Telegram ID игрока.
   * `username` (TEXT) — Игровой никнейм (вводится при создании персонажа).
   * `credits` (INT, DEFAULT 5000) — Стартовый баланс.
   * `ship_type` (TEXT, DEFAULT 'Старатель') — Начальный дирижабль/корабль.
   * `fuel` (INT, 100) | `x`, `y` (INT, 0) | `status` (TEXT, 'В порту').

2. **Таблица `locations` (Города-порты вселенной)**
   * Содержит координаты и описания. Стартовые порты:
   * `'Горн'` (0, 0) — Индустриальный центр.
   * `'Пар-Сити'` (400, 500) — Паровой мегаполис.
   * `'Ветроград'` (-200, 300) — Город парящих ветряков.

3. **Таблица `inventories` (Трюмы и Магазины)**
   * Поля: `owner_type` ('player' или 'location'), `owner_id`, `item_name`, `quantity`. Хранит все грузы и товары.

4. **🚀 ЗАМОРОЖЕННЫЙ СЛОЙ: Автономные Полетные Планы флота**
   * **Таблица `flight_plans`** — Хранит логические цепочки («если-то») для торговли в фоновом режиме.
   * **Таблицы `routes` и `route_points`** — Хранят навигационные трассы между городами.
   * *Статус:* Ждет подключения через отдельный слой `navigation.py`.

5. **Таблица `player_journals`** — Системный лог действий игрока.

### 🧠 ПАСПОРТ МОДУЛЯ: main.py (Умный Диспетчер Async)

* **Роль в игре:** Асинхронное ядро проекта. Управляет жизненным циклом бота, запускает сервер и распределяет входящие сообщения.
* **Принцип автоматизации (Как масштабируется):** 
  При старте сканирует корень проекта, находит все `.py` файлы (кроме системных) и загружает их в память. Если в файле есть метка кнопки, автоматически привязывает её к интерфейсу.
* **Связи с другими модулями:**
  * Запускает веб-сервер для Render через `server.start_hosting()`.
  * Берет конфигурацию бота и диспетчера из `bot_instance.py`.
  * Читает карту кнопок `GAME_BUTTONS` и собирает клавиатуру через `get_game_keyboard()` из модуля `interface.py`.
* **Правила безопасности:** Этот файл закрыт от изменений. Любая новая механика добавляется отдельным файлом без редактирования `main.py`.
