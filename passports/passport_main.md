# 📑 ТЕХНИЧЕСКИЙ ПАСПОРТ ЯДРА СИСТЕМЫ: main.py (OwnSky)

## 1. АРХИТЕКТУРНЫЙ ПАТТЕРН
* **Название**: «Умный Слепой Диспетчер» (на базе aiogram 3).
* **Суть**: Файл `main.py` является абсолютно «слепым». Он не содержит хардкода кнопок, команд, игровых сценариев или названий модулей. Его единственная задача — запустить инфраструктуру и автоматически собрать все части игры воедино.

## 2. МЕХАНИЗМ АВТОСКАНИРОВАНИЯ И СТЫКОВКИ Модулей
* **Логика работы**: При старте ядро рекурсивно обходит папку `handlers/` (включая любые подпапки) с помощью встроенных инструментов Python (`pkgutil.walk_packages`).
* **Правило стыковки (Правило Одной Строки)**: Чтобы ядро игры автоматически «увидело» любой новый игровой модуль, внутри этого модуля на уровне глобальной видимости обязательно должен быть объявлен объект роутера:
  ```python
  router = Router()
  ```
* **Регистрация**: Ядро динамически импортирует модуль, находит переменную `router` и выполняет команду `dispatcher.include_router(module.router)`. Порядок импорта не имеет значения.

## 3. ОКРУЖЕНИЕ И ЗАВИСИМОСТИ (Render)
* **Обязательные переменные среды (.env / Render Environment)**:
  * `BOT_TOKEN` — токен бота Telegram от BotFather.
  * `DATABASE_URL` — строка асинхронного/синхронного подключения к Neon.tech.
* **Поведение при ошибке**: Если хотя бы одна переменная отсутствует, ядро моментально завершает работу (`CRITICAL`), выводя понятную ошибку в логи Render.
* **База данных**: Инкапсулирована внутри модуля `database.py` (драйвер `pg8000`). Ядро не управляет пулом соединений хэндлеров; модули вызывают API базы данных напрямую.

## 4. ПРАВИЛО ИЗОЛЯЦИИ ИНТЕРФЕЙСА
* Ядро не занимается маршрутизацией кнопок. Нажатия на кнопки («В полет», «Трюм» и т.д.) перехватывают сами модули в папке `handlers/` через текстовые фильтры своего локального роутера.
